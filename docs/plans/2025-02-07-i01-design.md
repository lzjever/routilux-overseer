
### 1.2 设计原则

| 原则      | 应用                                     |
| --------- | ---------------------------------------- |
| **KISS**  | 不使用复杂的继承，用组合和服务层解决问题 |
| **DRY**   | QueryService 消除重复的 API 调用逻辑     |
| **SOLID** | 每层职责单一：UI → Service → Store → API |
| **YAGNI** | 不添加不需要的抽象，保持简洁             |

## 2. Service Layer 设计

### 2.1 APIClient 单例

**文件**: `lib/services/api-client.ts`

**职责**:

- 单例管理
- 自动注入 API Key
- 连接状态管理
- 避免 OpenAPI 配置重复设置

**关键接口**:

```typescript
class APIClientManager {
  static getInstance(): APIClientManager;
  configure(serverUrl: string, apiKey?: string): API;
  getAPI(): API;
  reset(): void;
  getConnectionInfo(): { serverUrl: string | null; apiKey: string | null };
}

export const apiClient = APIClientManager.getInstance();
export function configureAPI(serverUrl: string, apiKey?: string): API;
export function getAPI(): API;
export function resetAPI(): void;
```

### 2.2 QueryService

**文件**: `lib/services/query-service.ts`

**职责**:

- 封装常用查询逻辑
- 请求缓存（TTL: 30秒）
- 请求去重
- 统一错误处理

**关键接口**:

```typescript
class QueryService {
  query<T>(key: string, fetchFn: () => Promise<T>, options?): Promise<T>;
  invalidate(key?: string): void;

  flows: { list(); get(flowId); getMetrics(flowId) };
  jobs: { list(filters?); get(jobId); getMonitoringData(jobId) };
  workers: { list(filters?); get(workerId) };
  runtimes: { list(); get(runtimeId) };
  discovery: { syncFlows(); syncJobs() };
}

export const queryService = new QueryService();
```

## 4. WebSocket 优化

### 4.1 优化后的 WebSocketManager

**文件**: `lib/websocket/websocket-manager.ts`

**改进**:

- 指数退避重连（最大延迟 30 秒）
- 无限重连（可配置最大次数）
- 心跳检测（15 秒间隔，5 秒超时）
- 自动重订阅
- 离线/在线检测

**关键接口**:

```typescript
class WebSocketManager {
  connect(): Promise<void>;
  disconnect(): void;
  subscribeToJob(jobId: string): void;
  unsubscribeFromJob(jobId?: string): void;
  on(eventType: WebSocketEventType, handler: WebSocketEventHandler): () => void;
  isConnected(): boolean;
  getReconnectState(): { attempts: number; isReconnecting: boolean };
}

export function getWebSocketManager(
  serverUrl: string,
  config?: WebSocketManagerConfig
): WebSocketManager;
export function disposeWebSocketManager(): void;
export function initNetworkDetection(): void;
```

### 4.2 WebSocket Store

**文件**: `lib/stores/websocket-store.ts`

新增状态管理，用于跟踪 WebSocket 连接状态。

```typescript
interface WebSocketState {
  connected: boolean;
  reconnectAttempts: number;
  isReconnecting: boolean;
  setConnected(connected: boolean): void;
  setReconnectState(attempts: number, isReconnecting: boolean): void;
  reset(): void;
}
```

## 8. 风险与缓解

| 风险               | 影响 | 缓解措施                    |
| ------------------ | ---- | --------------------------- |
| API 客户端单例问题 | 高   | 充分测试，保留 reset() 方法 |
| Store 迁移破坏功能 | 高   | 逐个迁移，每个迁移后验证    |
| WebSocket 重连失败 | 中   | 保留旧的 manager 作为备份   |
| 测试覆盖率不足     | 中   | 渐进式提升，先核心后边缘    |
| 插件 UI 兼容性问题 | 低   | 插件系统架构不变，只加 UI   |

## 9. 架构改进对比

| 方面       | 改进前                    | 改进后            |
| ---------- | ------------------------- | ----------------- |
| API 调用   | 每次创建新实例            | 单例 + 缓存       |
| 状态管理   | 重复的 loading/error 逻辑 | QueryService 封装 |
| 错误处理   | 分散，不一致              | 统一 ErrorHandler |
| WebSocket  | 5 次重试                  | 无限重连 + 心跳   |
| 插件系统   | 无 UI                     | 完整管理界面      |
| 测试覆盖率 | < 30%                     | 目标 60%          |
| 代码重复   | 高                        | 低                |
