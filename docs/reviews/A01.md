## 0. 一句话定位

**Routilux-Overseer** 是一个基于 Next.js 14 的现代化 Web 应用，作为 Routilux 工作流框架的官方可视化管理与调试平台，当前处于**功能完整但需要架构优化**的阶段。

## 1. TL;DR 总结

### 总体健康度

- **架构成熟度**: 7/10 - 清晰的分层架构，但存在耦合问题
- **代码质量**: 7/10 - TypeScript 覆盖完整，但缺少集成测试
- **功能完整性**: 8/10 - 核心功能齐全，插件系统待完善
- **可维护性**: 6/10 - 状态管理分散，需要重构
- **生产就绪度**: 6/10 - 需要补充测试和监控

### 最大风险/技术债 TOP 3

1. **状态管理混乱** - 10+ Zustand stores 缺乏统一架构
2. **测试覆盖不足** - 仅有 4 个单元测试文件，无 E2E 测试
3. **API 客户端重复创建** - 每次调用都创建新实例，性能浪费

### 立即要做的 TOP 3

1. **重构状态管理层** - 统一 Store 架构，减少重复逻辑
2. **补充集成测试** - 至少覆盖核心 API 调用流程
3. **优化 API 客户端** - 实现单例模式，添加请求缓存

### 最容易被忽略但影响很大

**WebSocket 断线重连机制不够健壮** - 当前只有 5 次重试，生产环境可能导致数据丢失

### 2.2 关键执行路径

#### 流程 A: 连接到 Routilux 服务器

```
用户访问 /connect
  → connectionStore.setServerUrl()
  → connectionStore.setApiKey()
  → API 测试连接 (HealthService.readiness)
  → connectionStore.setConnected(true)
  → 跳转到 Dashboard
```

#### 流程 B: Flow 可视化查看

```
用户访问 /flows/{flowId}
  → flowStore.selectFlow()
  → API.flows.get(flowId)
  → convertFlowToNodes() / convertFlowToEdges()
  → layoutNodes() (Dagre 自动布局)
  → ReactFlow 渲染
```

#### 流程 C: Job 实时监控

```
用户访问 /jobs/{jobId}
  → jobStore.selectJob()
  → WebSocketManager.subscribeToJob(jobId)
  → 接收实时事件 (job_started, routine_completed, etc.)
  → jobEventsStore.addEvent()
  → UI 自动更新
```

**建议方案**:

```typescript
// 1. 创建通用 Store 基类
interface BaseStore<T> {
  data: Map<string, T>;
  loading: boolean;
  error: string | null;
  load: (serverUrl: string) => Promise<void>;
  clear: () => void;
}

function createBaseStore<T>() {
  return create<BaseStore<T>>((set, get) => ({
    data: new Map(),
    loading: false,
    error: null,
    load: async (serverUrl) => {
      set({ loading: true, error: null });
      try {
        const items = await get().fetchItems(serverUrl);
        set({ data: new Map(items.map((i) => [i.id, i])), loading: false });
      } catch (error) {
        set({ error: error.message, loading: false });
      }
    },
    clear: () => set({ data: new Map(), error: null }),
  }));
}

// 2. 各 Store 继承基类
export const useFlowStore = createBaseStore<FlowResponse>().extend((set, get, api) => ({
  fetchItems: async (serverUrl) => {
    const response = await createAPI(serverUrl).flows.list();
    return response.flows || [];
  },
  // Flow 特有方法
  selectFlow: async (flowId, serverUrl) => {
    /* ... */
  },
}));
```

**建议方案**:

```typescript
// lib/api/client.ts
class APIClient {
  private static instance: APIClient | null = null;
  private api: ReturnType<typeof createAPI>;

  private constructor(baseURL: string) {
    this.api = createAPI(baseURL);
  }

  static getInstance(baseURL?: string): APIClient {
```
